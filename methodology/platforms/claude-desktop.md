# Setting up with Claude Desktop

Claude Desktop is the simplest way to start a second brain if you're not comfortable with the terminal. You have two options: Projects (built-in) or a local folder with Claude Code.

## Option A: Claude Desktop Projects (simplest)

1. Open Claude Desktop
2. Create a new Project (click "Projects" in the sidebar)
3. Name it "My Brain" or "Second Brain"
4. In the Project's custom instructions, paste the contents of the CLAUDE.md template
5. Add your key files as Project Knowledge:
   - Start with `me/profile.md` and `me/thinking.md`
   - Add more files as your system grows

**Limitations:** Projects have a knowledge size limit. For a growing second brain, you'll eventually want to move to Claude Code or the MCP approach.

## Option B: Local folder + Claude Code (recommended)

This is the full-power approach. Your brain lives as files on your computer and Claude Code reads them directly.

1. Create a folder for your brain:
   - Mac: `~/Documents/brain/` or wherever you keep important files
   - Put it in a cloud-synced folder (iCloud, Dropbox) for backup

2. Install Claude Code (if you haven't):
   - Visit claude.ai/download and get Claude Code
   - It's a terminal app, but don't let that scare you: you just type in plain English

3. Open Claude Code in your brain folder:
   ```
   cd ~/Documents/brain
   claude
   ```

4. Tell Claude: "Read CLAUDE.md and help me set up my second brain"

5. Claude will read your instruction file and guide you from there

## Option C: Connect via MCP (for an assisted setup)

If you're connected to the Second Brain Starter MCP server, you already have access to the full methodology. Just say "Help me build my second brain" and follow the guided setup.

## Daily usage pattern

1. Start your AI session
2. The AI reads CLAUDE.md and orients itself
3. Work normally: discuss ideas, process meetings, draft documents
4. The AI updates your brain files as you work
5. Everything compounds for the next session

## Tips for non-technical users

- **Markdown is just text.** Files ending in `.md` are plain text with simple formatting. `# Heading`, `- bullet`, `**bold**`. That's 90% of what you need.
- **You don't need to organize perfectly.** Drop things in `inbox/` and let the AI sort them.
- **Talk to it like a person.** "I just had a meeting with Sarah about the Q2 plan. Here's what we discussed..." The AI handles the filing and integration.
- **It's OK to start messy.** The system self-organizes over time. Your first week will feel chaotic. By week three, it clicks.
