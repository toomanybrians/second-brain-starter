# Skill: Integrate Knowledge

## Triggers
"integrate this", "wire this in", "don't just file it"

## When to use
Automatically, whenever material is routed to `reference/`, `research/`, or `work/`. This is not optional.

## Procedure

1. **Read the material** and identify key insights, arguments, and data points
2. **Check active thinking** (`me/thinking.md`): does this strengthen, challenge, or add nuance to any current arguments or priorities?
3. **Check active work** (`work/` files): does this affect any current project?
4. **Check ideas** (`ideas/`): does this connect to any existing seeds?
5. **Update all affected files**: add references, update arguments, note connections
6. **Create new idea files** if the material introduces genuinely novel concepts
7. **Tag the source** with integration metadata: what was extracted, where it was wired

## Key principle

Knowledge that isn't connected to anything is noise. The value of new material comes from how it changes or enriches what you already know.
